# Final-Project

这是关于本项目的代码（基于深度强化学习的遥感卫星在线调度）。项目包含10个Python文件：

在整个项目代码中，对于A3C、DDPG、DQN和GA，我使用了一个框架，并根据研究的环境和问题进行了修改，其他文件都是我自己编写的。5个文件是关于深度强化学习算法的，3个是传统调度算法的，1个是环境文件，还有1个是用来帮助用户理解任务生成过程的文件。以下是每个文件的具体说明：

- **DRL_environment.py**：这是项目问题的环境文件，可用于训练A3C、DQN和DDPG，但在测试时需要将这些文件放在同一目录下。有一个重要的注意事项，因为研究需要测试不同的环境，所以在进行测试时，首先需要修改该文件中的任务数量，然后修改相关的深度强化学习算法文件中的任务数量，使其保持一致。如果要将深度强化学习算法与传统调度算法进行比较，也需要使任务数量保持一致。

- **深度强化学习算法文件**：
  - **DQN.py**：这是DQN的框架文件，结合具体的卫星问题，与环境进行交互。
  - **A3C.py, utils.py, shared_adam.py**：这三个文件用于构建A3C算法，与DQN一样，我对其进行了修改，以满足环境的需求。
  - **DDPG.py**：与A3C和DQN相同。

- **传统调度算法文件**：
  - **GA.py**：与深度强化学习算法类似，我对解码函数和主函数进行了修改，以适应该问题。
  - **FCFS.py**
  - **RP.py**

- **produce_tasks_test.py**：该文件可用于检查任务的生成情况，并检测任务冲突率。

如果要测试DDPG的适应性，需要首先使用两个不同的环境文件，然后对DDPG.py进行相应的修改。
